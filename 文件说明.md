# xbridge — 非侵入式定制 Bridge（基于 CCB）

## 这是什么

`xbridge` 是 [SeemSeam/claude_codex_bridge](https://github.com/SeemSeam/claude_codex_bridge)（CCB）的**定制 first-party 副本**，
功能与上游 **完全一致**（多 provider 同屏、`/ask` 跨 agent 路由、ccbd 守护、sidebar、Role Pack、worktree、热重载），
**唯一区别：彻底移除了对用户 tmux 配置的修改。**

- 上游版本溯源：**v7.6.12，commit `cf036b2`**
- 落地方式：copy 上游核心代码做 first-party（不用 submodule / 不用 docker，host 直装）

## 为什么做这版

上游 CCB 会接管用户的 tmux 会话主题（设 session 级 `status-style` / `status-format` /
`pane-border-status`、注册 `set-hook`、向 `~/.tmux.conf` 追加配置块），异常退出后还原不掉，
会出现「状态栏变灰、窗口列表被遮挡、点击切换失效」等问题。本版要求：**agent 照常在你现有
单 session 里以 window/pane 形式启动，但绝不改你的任何 tmux 配置。**

## 改了哪些（仅 4 处外科补丁，其余 100% 原样）

所有改动都用 `=== xbridge customization: NON-INVASIVE MODE ===` 注释标记，可全文搜索定位。

| # | 文件 | 上游行为（侵入） | 补丁 |
|:--|:--|:--|:--|
| 1 | `config/ccb-tmux-on.sh` | 运行时给当前 session 设 `status-style`/`status-format`/`window-status-format`/`pane-border-status` + `set-hook` | 守卫后 `exit 0`，默认不执行 |
| 2 | `config/ccb-tmux-off.sh` | 还原上述选项 | `exit 0`（on 没动过，无需还原） |
| 3 | `install.sh` `install_tmux_config()` | 向 `~/.tmux.conf`（或 `.local`）追加 CCB 配置块 | helper 脚本照装，`return 0` 跳过所有 `~/.tmux.conf` 写入 |
| 4 | `lib/cli/services/tmux_ui_runtime/service.py` `apply_project_tmux_ui()` | **Python 旁路**：直接 `set-option` session 主题 + 全局 root 鼠标绑定 + `set-hook` + pane 边框主题 | 函数入口守卫早退，一处覆盖全部 4 项 session 改动 |

### 行为开关
- 默认（`XB_TMUX_THEME` 未设或 ≠ `1`）：**非侵入模式**，不碰任何 tmux 配置。
- `export XB_TMUX_THEME=1`：**恢复上游主题行为**（如果哪天你又想要那套配色/边框标签）。

### 取舍说明
非侵入模式下失去的仅是 **tmux 视觉层**：状态栏配色、pane 边框上的 agent 名标签、sidebar 随窗口
自动 resize 同步。agent 身份仍可通过 **window 名 + sidebar pane** 识别；所有协作/通信功能不受影响。

## 安装 / 使用 / 卸载

```bash
# 安装（host 直装，会创建 ~/.local/bin/ccb 等；不会改 ~/.tmux.conf）
cd ~/projects/xbridge && ./install.sh

# 在某个项目里用（命令仍为 ccb，内部命名保持上游以确保功能一致）
cd /your/project
ccb config validate
ccb                      # 在当前 tmux session 起 agent 工作台
ccb ask <agent> "<task>" # 跨 agent 路由（--callback/--silence/--artifact）
ccb reload               # 热加载配置
ccb kill                 # 停后台运行时
ccb uninstall            # 卸载（因从不写 ~/.tmux.conf，无残留需清）
```

> 命令名仍是 `ccb`：内部大量引用该名，重命名风险高、会破坏「功能一致」目标，故保留。
> 如需改成独立命令名（避免与上游共存冲突），属后续可选项，单独处理。

## 验证非侵入（安装后自检）

```bash
# 装前先记录指纹，装后比对——应完全一致
md5 ~/.tmux.conf
# 起一次 ccb 后，确认 session 没有被设主题：
tmux show -gv status-style        # 应为你自己的值，不是 bg=#1e1e2e
tmux show 'status-format[0]'      # 应为空（无 session 级覆写）
```

## 与上游同步

需要跟进上游更新时：重新 clone 上游 → 比对本文件「改了哪些」表中的 4 个文件 → 重打补丁即可。
补丁点都有 `xbridge customization` 标记，迁移成本极低。
